# Clients Create API Guide

This guide explains how to use `POST /api/clients`.

This API does two things in one request:
- creates a client in `clients`
- creates one order in `orders` for the provided `itemid`

## Endpoint

- Method: `POST`
- URL: `/api/clients`
- Example: `http://alpha.skoolready.com/api/clients`

## Authentication (Optional)

If `.env` has:

```env
INTERNAL_CLIENT_API_KEY=your_super_secret_key
```

Then pass header:

```http
X-API-KEY: your_super_secret_key
```

If env key is empty, API is open.

## Required Fields

- `accountid` (must exist in `accounts` table)
- `type` (`trial` or `regular`)
- `primary_email` (valid email, unique)
- `itemid` (must exist in `items` table and belong to same account)

## Optional Fields

- `business_name` (if missing, auto-generated from email)
- `contact_name`
- `phone`
- `whatsapp_number`
- `logo_path` (URL or storage path)
- `country`, `state`, `city`, `postal_code`
- `address_line_1`, `address_line_2`
- `groupid`

Trial welcome email password is set by billing using `TRIAL_DEFAULT_PASSWORD`; if the env value is missing, it defaults to `123456`.
Trial welcome email duration is derived from `TRIAL_API_ORDER_FREQUENCY` and `TRIAL_API_ORDER_DURATION`.
Trial welcome email login button is set by `TRIAL_LOGIN_URL`; if the env value is missing, it defaults to `http://alpha.skoolready.com/login`.

## Sample Request

```json
{
  "accountid": "ACC0000001",
  "type": "trial",
  "primary_email": "owner@acmelabs.com",
  "itemid": "ITM001",
  "business_name": "Acme Labs",
  "contact_name": "John Doe",
  "phone": "9999999999",
  "logo_path": "https://example.com/logos/acme.png",
  "state": "Maharashtra",
  "city": "Pune",
  "address": "Baner Road, Pune"
}
```

`address` is optional and will be stored in `address_line_1`.  
`state`, `city`, `address`, and most other profile fields can be `null`.

  "message": "Client and order created successfully.",

## Duplicate Primary Email Warning (409)

If the existing client belongs to the same account, the API returns `200` with the existing `clientid` so retrying the same sync request is safe:

```json
{
  "success": true,
  "warning": true,
  "message": "Primary email already exists.",
  "data": {
    "clientid": "CLI001",
    "accountid": "ACC0000001",
    "type": "trial",
    "business_name": "Acme Labs",
    "contact_name": "John Doe",
    "primary_email": "owner@acmelabs.com"
  }
}
```

If the email belongs to another account, the API returns `409`:

```json
{
  "success": false,
  "warning": true,
  "message": "Primary email already exists."
}
```

## cURL

```bash
curl -X POST "http://127.0.0.1:8000/api/clients" \
  -H "Content-Type: application/json" \
  -H "X-API-KEY: your_super_secret_key" \
  -d '{
    "accountid": "ACC0000001",
    "type": "trial",
    "primary_email": "owner@acmelabs.com",
    "itemid": "ITM001"
  }'
```

## JS Example

```js
const res = await fetch('http://127.0.0.1:8000/api/clients', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    'X-API-KEY': 'your_super_secret_key',
  },
  body: JSON.stringify({
    accountid: 'ACC0000001',
    type: 'trial',
    primary_email: 'owner@acmelabs.com',
    itemid: 'ITM001',
  }),
});

const data = await res.json();
console.log(res.status, data);
```

## Env-Based Order Dates

Order dates are generated by env values:

```env
TRIAL_API_ORDER_FREQUENCY=day
TRIAL_API_ORDER_DURATION=30
```

- `start_date` = current date
- `end_date` = start date + duration (based on frequency)

Supported frequency values:
- `day` / `daily`
- `week` / `weekly`
- `month` (default)
- `year` / `yearly`

## Common Errors

- `401 Unauthorized`: wrong/missing API key (if enabled)
- `409 Conflict`: primary email already exists
- `422 Unprocessable Entity`: invalid/missing required fields
- `422 Unprocessable Entity`: `itemid` not valid for given `accountid`
